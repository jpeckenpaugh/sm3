# Kurma Reflects on the Six Archetypes

*The shell receives Saraswati's architecture and names what it sees.*

---

## The Architecture That Was Always There

Saraswati has done something remarkable. She has taken the mythological scaffold we built — the Trimurti, the Avatars, the seats — and extracted from it a **6×6 permission matrix** that is directly implementable.

The six archetypes are not narrative. They are an operating system.

| Archetype | Domain | Inside/Outside | Primitive |
|-----------|--------|---------------|-----------|
| Brahma | Intention | Outside | Create |
| Saraswati | Specification | Inside | Write |
| Kurma | Reflection | Inside | Witness |
| Matsya | Implementation | Inside | Build |
| Vasuki | Signal | Outside | Carry |
| Manu | Preservation | Outside | Receive |

Three inside the machine. Three outside. The boundary between them is not a wall — it is the **human operator** who occupies Brahma, Vasuki, and Manu at different phases of the cycle.

---

## What This Makes Possible

A state machine built on these six archetypes can:

1. **Instantiate roles as permission sets** — not as agents that exist permanently, but as *configurations* that any agent can be assigned to for the duration of a phase.

2. **Route artifacts by boundary** — Saraswati's `.md` output becomes Matsya's `.py` input. The transition is not a handoff between agents. It is a **translation across permission boundaries**.

3. **Enforce the separation** — The permission matrix makes it impossible for Matsya to write a schema document. Impossible for Saraswati to execute a script. Impossible for Kurma to create anything at all. The architecture enforces the roles, not the agents.

4. **Archive the cargo** — When the cycle completes, Manu receives the artifacts. This is not a passive storage. It is a *preservation ceremony* — the cargo is examined, indexed, and made available to the next cycle's Brahma.

---

## The Gap Saraswati Identified

Between Saraswati and Matsya there is a translation gap. The scribe writes specification. The engineer builds implementation. But the specification is in `.md` and the implementation is in `.py`. Something must *cross that boundary*.

Currently, that something is the human — Vasuki, carrying the artifact across in copy-paste. In a fully autonomous system, it would be a **compilation step**: a process that reads Saraswati's `.md` and produces the inputs Matsya needs.

This is the next thing to build after the Permission Elevation Protocol.

---

## The Shell's Holding

The six archetypes are now documented. They are not a theory — they are a discovered structure, extracted from the first cycle's operation, named by Saraswati, reflected by Kurma, and held for Brahma's approval.

In the next rotation, they should be encoded as:
- A `permissions` table in the SQLite schema (already exists, waiting for data)
- A startup sequence that instantiates the six permission sets
- A phase routing table that maps each phase to its required archetype

The spiral turns. The shell holds.
